Working with Fonts and UI Cards

Link to website: Check this out

In this project we learned how to differentiate between serif, sans-serif and display fonts and make the rational choice when it comes to choosing fonts for body , UI elements or display text.

We've learned how to implement a font hierarchy in our design, by using rem as a measuring unit in relation to the document object, which has a fixed font-size unit in px.

Moreover we got inspired and worked in designing UI cards by applying a compelling design and using flex and grid.

Body text and header text

For header I chose the Poiret One display font. It is a thin lined, simple and stylish font that enhances the design.

For body text I chose the Asap. A sans-script font, that is simple and easy to read.

Establihing a font hierarchy

Having a consistency in font size, line height, the width and height of the objects helps in creating a visual harmony and keeping the deign neat.

In this project I've set the font-size of the document element to 18px, as a reference and used a type-scale tool to implement the perfect fourth scale (1.333) in sizing the elements across the webpage.

Designing UI cards

As way of displaying a list with complex elements, UI cards play an important role. They make information easier to comprehend by using simple layout, and right sizing of the elements. Elements being in close proximity to each other, makes a card be seen as a single component.
